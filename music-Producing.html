Fundamentos Técnicos

El Lenguaje de los Comandos

He aprendido a simplificar conceptos técnicos para recordarlos mejor.

  • `href`: Es la **dirección de destino**. Es a dónde vamos.
  • `src`: Es la **fuente de origen**. Es de dónde viene el archivo.
  • `class`: Es la **etiqueta para estilos**. Permite aplicar un diseño (CSS) a múltiples elementos.

Procesando el Sonido y el MIDI

La clave no es el audio en sí, sino cómo se manipulan sus variables. Los programas no guardan un archivo para cada nota, sino que procesan la frecuencia (Hertz) de un solo sonido para crear toda la escala. Esto se logra con la **Web Audio API**.

// Ejemplo de cómo la lógica transforma la frecuencia (Hertz) en notas musicales

// Las notas musicales se basan en frecuencias (Hertz). const notasMusicales = { 'C4': 261.63, 'D4': 293.66, // ... y así sucesivamente para las 12 notas }; // La Web Audio API nos permite cambiar la frecuencia de un sonido. // Esto es lo que hace un sintetizador. const oscillator = audioContext.createOscillator(); oscillator.frequency.value = notasMusicales['C4']; // Cambia el valor para tocar otra nota

Paso 2: Proteger tu Trabajo y tu Marca

Tu idea es tu activo más valioso. Debes protegerla con **propiedad intelectual** (derechos de autor, patentes, marcas). Es crucial registrar tu marca en el **IMPI** y ser cuidadoso al compartir ideas públicamente.

Paso 3: El Despliegue Profesional

Para que tu proyecto esté en línea, necesitas un servicio de hosting. Opciones como **GitHub Pages** (gratuito) o **Firebase Hosting** (ideal para escalar) son perfectas para mostrar tu portafolio y tu conocimiento de los flujos de despliegue profesionales.
